# ADRs

Architecture Decision Records capture decisions that future contributors and agents should not re-litigate without new evidence.

## Naming

Use sequential files:

```text
0001-short-kebab-title.md
0002-short-kebab-title.md
```

## Status values

- `Proposed`
- `Accepted`
- `Superseded by ADR-000N`

## Template

```md
# ADR-000N: Title

Date: YYYY-MM-DD
Status: Proposed | Accepted | Superseded by ADR-000N

## Context

What forces, constraints, or repeated confusion led to this decision?

## Decision

What are we choosing?

## Consequences

What gets easier, what gets harder, and what must future work respect?

## Alternatives considered

- Alternative: why rejected.
```
