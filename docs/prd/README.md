# PRDs

Product Requirements Documents describe user-visible behavior, constraints, and acceptance criteria before or during implementation.

Use PRDs when a change has product ambiguity, multiple interaction states, or enough scope that tests alone will not communicate intent.

## Naming

Use date-prefixed files:

```text
YYYY-MM-DD-short-kebab-title.md
```

## Template

Copy `TEMPLATE.md` for new PRDs.
